-git status verifica os arquivos que ainda não foram incluídos no controle de versão.
-git add . -> adiciona ao controle de versão.
-git commit - m "comentario"
-git push -f origin master //Com a tag -f você irá substituir o conteúdo da branch remota          pelo código que você tem no seu repositório local//
-git push -u origin main // puxa tudo que esta no repositório local para o remoto na branch especificado no caso(main)

* git reflog -> verificar o histórico de versões.

* git reset --hard "id"  ->voltar a uma versão do código.

# Branch
git branch -> verifica as branchs que estão disponíveis no projeto.

git branch "nome da branch -> cria uma nova branch 
git checkout "nome da branch" -> altera para branch a ser usada.

Antes de fazer o marge entre na branch master e rode >git pull para atualizar a última versão do código.
git merge "nome da branch" -> puxa as atualizações para a branch que se deseja unir os códigos.

git pull atualiza a última versão do código.
git checkout -b "nome da branch" master -> cria uma nova branch com base na branch master.

git log -> mostra informações dos últimos commits
